* !DOCTYPE html은 HTML이 어떤 버전으로 작성되었는지 브라우저에게 알려주는 것으로,
  버전차이에 따른 호환성을 높이기 위해 사용되는 선언문입니다. (!DOCTYPE html은 HTML5 버전)

* html은 HTML 문서를 구성하는 기본 단위로, 문서의 루트 요소(root element)를 정의할 때 사용되는 함수입니다.
  (웹 페이지의 언어를 선언할 때 항상 html 태그 안에 lang속성을 설정해야 됩니다.)

* head는 문서의 머리를 나타내는 태그로 브라우저 화면에 직접적으로 보이지 않으며,
  숨은 데이터를 정의하는 태그들이 들어가게 됩니다. (title, meta, style, base, link, script, noscript 등등)

* body는 글의 본문같은 것으로 해당 HTML 문서의 텍스트, 하이퍼링크, 이미지, 리스트 등과 같은 모든 콘텐츠를 포함하는
  영역을 정의할 때 사용합니다. HTML 문서에는 단 하나의 body요소만이 존재할 수 있습니다.

* meta는 base, link, script, style, title 태그와 같은 메타데이터 관련 태그들이 나타낼 수 없는 다양한 종류의
  메타데이터를 제공할 때 사용되며, 제공된 정보는 브라우저나 검색엔진, 다른 웹 서비스에서 사용하게 됩니다.
  (charset="UTF-8" = character(문자)+set(규칙) "UTF-8 형식의 규칙대로 문자를 읽어라")
  (meta태그는 언제나 head태그 내부에 위치해야 됩니다.)

* strong은 글씨를 짙게 강조하고 싶을 때 사용하는 태그입니다.

* h1은 제목과 같은 효과를 지닌 태그로 h1 ~ h6까지 있고, 1에서부터 6으로 갈 수록 글자 크기가 작아집니다.

* u는 underline(밑줄)을 치고 싶을 때 사용하는 태그입니다.

* br은 입력 시 br사이의 문자를 줄바꿈 해주는 태그로
  줄바꿈이라는 시각적 정보 외엔 기능이 없기 때문에 따로 마치는 태그가 없습니다.

* p는 입력 시 태그의 시작과 끝 사이에 있는 문자의 단락을 나눠주는 태그로
  br과는 다소 활용의 차이점이 있는 태그이며, css의 margin을 이용해 줄바꿈 사이의 폭을 조절할 수 있습니다.

* nbsp는 'non breaking space'의 약자로 여러 개의 공백을 하나의 공백으로만 인정하는
  html에서의 공백을 채워주는 특수문자입니다.

* 이미지 파일의 사이즈 조절을 할때 width와 height로 하나부터 조정할 수 있고,
  width나 height에서 0~100%(보고있는 사이트 화면크기를 기준으로 채워지는 너비, 높이의 비율)단위로 설정할 수 있습니다.

* li는 'list'의 약자로 목록을 만들 때 쓰이는 태그이며, ul태그의 자식태그입니다.

* ul은 'unorder list'의 약자로 순서가 없는 HTML 리스트를 만들 때 쓰이는 태그이며, li태그의 부모태그입니다.

* ol은 'ordered list'의 약자로 숫자나 알파벳 등 순서가 있는 HTML 리스트를 만들 때 쓰이는 태그이며, li태그의 부모태그입니다.
  ul태그와 차이점을 본다면 위에 적은 목록의 순서에서 첫번째 순서의 HTML을 뺐을 때 ul은 CSS부터 C++까지
  매겨진 번호를 1~5로 모두 고쳐야 되는 반면, ol은 HTML을 빼도 자동으로 CSS가 1번이 되며 순서가 모두 정렬이 됩니다.

* a는 'anchor'의 약자이자 'Hypertext'가 의미하는 태그로, 다른 페이지를 연결할 때 쓰는 하이퍼링크를 정의할 때 사용됩니다.
  만약 링크로 이동할 때 기존의 페이지가 아닌 새 탭을 열어 이동하고 싶다면, 속성 'target'에서
  속성 값 '_blank'를 입력하면 됩니다.(앞에 언더바 입력 필수)
  또 마우스를 링크에 갖다 댔을 때, 어떤 사이트인지, 미리 툴팁으로 확인시키고 싶을 땐 title 속성을 써서
  설명을 적어주면 됩니다.
